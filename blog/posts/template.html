HTTPS & SSL Certificates — What I Finally Understood

I've known about HTTP vs HTTPS for years. Today I actually understood it. Here's what finally clicked about encryption, verification, and why hackers can't fake certificates.

The Problem HTTPS Solves

HTTP sends data in plain text. Anyone in the middle (WiFi owner, ISP, hacker) can read your passwords, messages, everything.

HTTPS encrypts all data. Even if intercepted, it's unreadable garbage. But encryption alone isn't enough. You also need to verify you're talking to the real website, not a hacker pretending to be that website.

This is where SSL certificates come in.

How Signing Works

When a Certificate Authority (like Let's Encrypt, DigiCert, or Cloudflare) signs a website's certificate:

  • They hash the certificate content
  • Encrypt that hash with their private key
  • Attach it as a signature

Only the CA can create this signature. Because only they have the private key.

How Your Browser Verifies

When your browser receives a certificate:

  • Decrypts signature using CA's public key
  • Hashes the certificate content independently
  • Compares both hashes

If they match = genuine certificate. If not = tampered or fake.

The Key Insight

Hackers have the public key too. Everyone does. But public keys can only VERIFY signatures, not CREATE them. To create a valid signature, you need the private key. Only the CA has it.

That's the beauty of asymmetric cryptography. The public key being public isn't a weakness. It's designed that way.

Code Example

Here's how you might verify a certificate in code: 

# Pseudo-code for certificate verification
signature = certificate.signature
content = certificate.content

# Decrypt signature with CA's public key
decrypted_hash = decrypt(signature, ca_public_key)

# Hash the content independently
calculated_hash = hash(content)

# Compare
if decrypted_hash == calculated_hash:
    print("Certificate is genuine!")
else:
    print("Warning: Certificate tampered!")

Key Takeaway

Sometimes you use something daily without understanding how it works. Today I fixed that for HTTPS.

The whole security model comes down to one thing: private keys CREATE, public keys VERIFY. Without the private key, you can't forge anything.
